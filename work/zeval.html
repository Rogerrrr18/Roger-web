Zeval

Agent 产品大规模落地之后，没有人能真正看清它在生产环境里做对了什么、做错了什么。现有工具要么只告诉你系统有没有崩，要么只能跑离线 Benchmark——真实业务里持续追踪交互质量的在线评测，几乎是空白。

Zeval 构建的是一套 AI Agent 效果诊断与质量运营系统。接入对话日志，自动发现摩擦点，生成回归用例，让 Agent 迭代第一次能用数据说话。

问题

Agent 产品上线后，团队普遍掉进同一个坑：人工抽听对话成本极高，漏掉的比发现的多；让 LLM 自动打分，指标又和业务目标对不上；埋点数据只能看「有没有用」，看不到「用得好不好」。

结果是大量 Agent 在版本迭代中看不见问题，用户体验持续恶化，死在优化跟不上的地方。

核心能力

  • 隐式推断评测 — 不要求用户打分。分析用户后续行为自动推断满意度：沉默 = 接受，追加纠正 / 重复提问 = 不满
  • 恢复轨迹识别 — 自动标记「先失败 → Agent 纠偏 → 最终成功」的对话片段，这类数据是改进 Agent 最高价值的素材
  • 交互摩擦点发现 — 自动识别对话断点、用户流失节点、目标未完成的轮次
  • 评测数据飞轮 — 生产失败 → 自动生成回归用例 → 入库 → 发版前自动跑回归
接入日志→ 发现问题→ 生成用例→ 回归验证→ 产品迭代

市场

  • 57% 的企业已将 Agent 部署到生产环境，质量问题是最大的部署障碍
  • LLM 可观测性平台市场：2026 年约 $27 亿，年增速 36%
  • 现有工具（Langsmith、Braintrust）定位通用 trace 查看，缺少业务目标对齐与隐式推断能力

商业模式

SaaS 订阅 + 私有化部署。数据主权优先：客户数据始终归客户所有，Zeval 以 SDK 形式接入，处理在客户自己的基础设施上完成。

冷启动路径：先与 1–3 家头部 AI 产品团队深度共生，免费跑完整 eval 闭环，换真实场景打磨机会，再以案例为锚点横向复制。
